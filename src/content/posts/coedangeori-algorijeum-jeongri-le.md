것이고, 더욱 효율적인 알고리즘이 있을 수 있습니다.</p><p>​</p><p>​</p><p>​</p><h2 id="1-bfsbroad-first-search"><strong>1. BFS(Broad-First-Search)</strong></h2><p>현재 노드에 연결된 간선을 Queue에 쌓아 놓고, First In First Out 방식으로 노드를 돌며 전체탐색하는 알고리즘 입니다.</p><p>​</p><p>"우리는 단지 탐색만 하는 것이 아니라 최단거리를 찾기 위한 BFS가 필요합니다."</p><p>​</p><p><strong>0) 따라서 먼저 모든 노드의 거리 값에 INF값을 넣고, 시작노드의 거리값은 0으로 초기화 합니다.</strong></p><p>그 후 연결된 간선을 확인할 때, <strong>"현재 노드까지의 거리 + 1"</strong> 과, <strong>"연결노드에 저장된, 해당 노드까지의 거리"</strong>를 비교하여 더 작은 것을 저장하고, 이어서 탐색하기 위해 해당 노드를 enQueue하면 됩니다.</p><p>​</p><p>1) <strong>최단거리 알고리즘으로서 BFS를 사용하기 위해서는, 간선의 가중치가 모두 같을 때(ex) w=1)에만 사용할 수 있습니다.</strong></p><p>​</p><p>2) <strong>간선에 연결된 노드가 이미 방문한 노드일 가능성이 있으므로 방문체크를 해주어야 합니다.</strong></p><p>사실 0)의 과정만 수행해도 방문처리는 되지만, TLE 혹은 Stack-Overflow가 날 가능성이 있으므로 연결된 노드에 저장된 거리가 INF일 때만 탐색을 진행하면 됩니다.</p><p>​</p><p><strong>3) 시간복잡도는 인접리스트 기준 O(V+E), 인접행렬 기준 O(V^2) 입니다.</strong></p><p>​</p><p>순서대로 정리하면</p><p>1. 시작 노드를 Queue에 넣습니다.</p><p>2. Queue.front()를 현재 노드 current 라 하고, Queue.pop()합니다.</p><p>3. current에 연결된 노드들 중, 아직 방문되지 않은 노드 next 를 확인합니다.</p><p>4. "current까지의 거리+1"와 "next에 저장된 거리" 중 작은 것을 next에 저장하고 next를 push합니다.</p><p>5. 2~4과정을 Queue가 빌 때 까지 반복합니다.</p><p>​</p><p>결과 : 모든 노드에 시작 노드로부터의 최단거리가 저장됩니다.</p><p>​</p><p>참고 : DFS는 최단경로의 보장이 되지 않습니다.</p><p>​</p><p>​</p><h2 id="2-0-1bfszero-one-broad-first-search"><strong>2. 0-1BFS(Zero-One Broad-First-Search)</strong></h2><p>BFS와 동일하나, 가중치가 0, 혹은 1 뿐인 그래프에서 사용할 수 있는 최단거리 알고리즘입니다.</p><p>탐색을 위해 Queue가 아닌 Deque를 사용합니다.</p><p>​</p><p><strong>1) 상술하였듯, 그래프의 가중치가 0, 혹은 1일 경우에만 사용할 수 있습니다.</strong></p><p>​</p><p><strong>2) 다음 노드로 가는 가중치가 0이면 Deque의 front에, 1이면 Deque의 back에 push합니다.</strong></p><p>"최단거리 알고리즘"이기 때문에, 가중치가 작은 0인 간선을 우선적으로 탐색하여야 합니다.</p><p><strong>​</strong></p><p><strong>3) 이외의 사항은 일반적인 BFS와 동일합니다.</strong></p><p>​</p><p>순서대로 정리하면</p><p>1. 시작 노드를 Deque에 넣습니다.</p><p>2. Deque.front()를 현재 노드 current 라 하고, Deque.pop_front()합니다.</p><p>3. current에 연결된 노드들 중, 아직 방문되지 않은 노드 next 를 확인합니다.</p><p>4. <strong>"(current까지의 거리)+(next로 가는 가중치)"</strong>와<strong> "next에 저장된 거리"</strong> 중 작은 것을 next에 저장합니다.</p><p><strong>5. next로 가는 가중치 w가 0이면 front에 push하고, 아니면 back에 push합니다.</strong></p><p>6. 2~5과정을 Deque이 빌 때 까지 반복합니다.</p><p>​</p><p>결과 : 모든 노드에 시작 노드로부터의 최단거리가 저장됩니다.</p><p>​</p><p>​</p><p>​</p><h2 id="3-%EA%B0%9C%EC%84%A0%EB%90%9C-%EB%8B%A4%EC%9D%B5%EC%8A%A4%ED%8A%B8%EB%9D%BC-%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98-improved-dijkstra-algorithm"><strong>3. 개선된 다익스트라 알고리즘 (Improved Dijkstra Algorithm)</strong></h2><p>유명한 최단거리 알고리즘입니다. Dynamic Programming과 Greedy를 활용한 알고리즘으로 최단거리 테이블에 <strong>시작노드로부터 다른 모든 노드로 가는 최단거리</strong>를 저장합니다.</p><p>​</p><p><strong>시작노드로부터 대상 노드로 가는 최단거리를 찾을 때, <u>최소 가중치를 가지는 간선을 선택</u>하여 해당 간선을 거쳐 대상노드로 가는 최단거리를 찾게되고, 이 과정을 노드의 개수 V만큼 반복하게 됩니다.</strong></p><p><strong>​</strong></p><p><strong><u>최소 가중치를 가지는 간선을 선택</u></strong>할 때, 가고자 하는 노드까지의 최소 가중치를 선형 탐색으로 찾을 수도 있습니다.(기본 다익스트라의 경우) 다만 이 경우 시간복잡도는 O(VE)가 나오게 되고 PS문제에서의 시간 제한으로 인해 TLE가 나오는 경우가 다수 생깁니다.</p><p>​</p><p>따라서 <strong>개선된 다익스트라 알고리즘</strong>에서는, 최소 가중치 간선을 선택할 때<strong> Priority Queue(Heap)를 사용</strong>하여 로그시간에 최소 가중치를 찾게 되고, 이 경우 <strong>시간복잡도는 O(ElogV)</strong>로 상당히 개선됩니다.</p><p><strong>​</strong></p><p><strong>*다익스트라 알고리즘은 음수 사이클이 존재하는 그래프에서의 최단거리가 보장되지 않습니다.</strong></p><p>=&gt; 음수 사이클 검출 가능한 벨만-포드, 플로이드-워셜 사용</p><p>​</p><p>​</p><p>순서대로 정리하면</p><p><strong>0. 시작노드로부터 각 노드로 향하는 최단 거리 테이블 d를 생성하고, INF로 초기화 합니다.</strong></p><p>- 2차원 인접 행렬/리스트의 경우 1차원 dynamic table이 필요합니다.</p><p>- 시작노드의 거리값은 0으로 초기화 해둡니다.</p><p>1. 시작 노드를 최소 힙에 넣습니다.</p><p>2. 힙의 루트노드를 현재 노드 current 라 하고, 힙을 pop합니다.</p><p>3. current로 가는 가중치가 이미 d[current]보다 크면 2.로 돌아갑니다.</p><p>4. current에 연결된 노드들 중 하나인 next 를 확인합니다.</p><p><strong>5. "d[current] + current에서 next로 가는 거리"와 "d[next]" 중 작은 것을 d[next]에 저장하고 이때 해당 간선의 정보를 push합니다.</strong></p><p>6. 2~5과정을 힙이 빌 때 까지 반복합니다.</p><p>​</p><p>*5.의 과정에서 로그시간만에 항상 최소인 간선만을 선택하게 됩니다.</p><p>​</p><p>결과 : 시작노드로부터 모든노드로 가는 최단거리가 d에 저장됩니다.</p><p><strong>​</strong></p><p><strong>​</strong></p><p><strong>​</strong></p><h2 id="4-%EB%B2%A8%EB%A7%8C-%ED%8F%AC%EB%93%9C-%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98bellman-ford-algorithm"><strong>4. 벨만-포드 알고리즘(Bellman-Ford Algorithm)</strong></h2><p>다익스트라와 비슷하나, <strong>음수 간선 및 사이클이 존재할 때에도 사용할 수 있습니다.</strong></p><p>다만, 다익스트라와 다르게 반복할 때마다 모든 간선을 확인하여 안정화하기 때문에 <strong>시간복잡도는 O(VE)로 다익스트라보다 느립니다.</strong></p><p>​</p><p><strong>1) 최단거리 테이블의 안정화는 V-1번째에 끝납니다.</strong></p><p>V개의 정점에 대하여 사이클이 없는 최대 간선의 개수는 V-1개이기 때문입니다.</p><p>​</p><p><strong>2) V번째 안정화에서 최단거리 테이블의 갱신이 이루어진다면 음수 싸이클(Negative Cycle)이 존재한다는 것입니다.</strong></p><p>​</p><p>순서대로 정리하면</p><p><strong>0. 시작노드로부터 각 노드로 향하는 최단 거리 테이블 d를 생성하고, INF로 초기화 합니다.</strong></p><p>- 2차원 인접 행렬/리스트의 경우 1차원 dynamic table이 필요합니다.</p><p>- 시작노드의 거리값은 0으로 초기화 해둡니다.</p><p>1. 간선 e에 대해 "<strong>d[출발노드]+e의 가중치</strong>" 와 "<strong>d[도착노드]"</strong> 중 작은 것을 d[도착노드]에 저장합니다.</p><p>2. 이번 수행이 V번째 수행이고 d[도착노드]가 변경되었다면 음수 사이클이 존재하는 것입니다.</p><p>3. 1~2까지의 과정에서 확인하지 않은 간선이 있다면 1.로 이동해 다음 간선을 확인합니다.</p><p>4. 1~3의 과정을 V-1번 (음수 사이클 유무를 알고 싶다면 V번) 반복합니다.</p><p>​</p><p>결과 : 모든 노드에 시작 노드로부터의 거리가 저장됩니다. / 음수 사이클의 존재여부를 알 수 있습니다.</p><p><strong>​</strong></p><p><strong>​</strong></p><p><strong>​</strong></p><h2 id="5-%ED%94%8C%EB%A1%9C%EC%9D%B4%EB%93%9C-%EC%9B%8C%EC%85%9C-%EC%95%8C%EA%B3%A0%EB%A6%AC%EC%A6%98-floyd-warshall-algorithm"><strong>5. 플로이드-워셜 알고리즘 (Floyd-Warshall Algorithm)</strong></h2><p>다익스트라, 벨만포드가 한 노드로부터 다른 모든 노드로의 최단 거리를 구했다면,</p><p>플로이드-워셜 알고리즘은 <strong>모든 노드로부터 모든 노드로의 최단 거리</strong>를 구하는 알고리즘입니다.</p><p>​</p><p>다익스트라 혹은 벨만포드로 모든 노드로부터 모든 노드로의 최단거리를 구하게 되면 시간복잡도는 각각 O(V*ElogV), O(V^2 * E)로 플로이드 워셜보다 느립니다.</p><p>​</p><p>플로이드-워셜은 간선이 없더라도 어떤 노드로부터 <strong>어떤 노드를 거쳐</strong> 어떤 노드로 가는 거리를 <strong>모두 확인</strong>하게 됩니다. 따라서 <strong>시작노드, 경유노드, 도착노드를 전부 확인하기 때문에 O(n^3)이라는 긴 시간복잡도를 갖게 됩니다.</strong></p><p>​</p><p><strong>벨만-포드와 동일하게, 음수 간선 및 사이클이 존재할 때에도 사용할 수 있습니다.</strong></p><p>​</p><p><strong>1) 인접행렬을 사용하는 것이 편합니다.</strong></p><p><strong>​</strong></p><p><strong>2) 출발노드와 도착노드가
